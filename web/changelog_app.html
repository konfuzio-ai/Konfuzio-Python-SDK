Changelog¶

All notable changes in the Konfuzio Server will be documented according to the principles defined by Keep a Changelog.

The changelog adheres to Calendar Versioning and the release tag relates to the date and time when those changes have been released to app.konfuzio.com.

Self-hosted Konfuzio Server can be upgraded according to the documentation.

Planned¶

You can think of the Planned section as a Roadmap that lists Konfuzio Server features our team is actively working on. This list covers a planning horizon of 12 weeks.

  • Add self-service customer center to manage contracts and invoices (Internal Documentation, Internal Ticket).

  • Calculate and access Tokenizers via the web interface. The name Tokenizer might be revised. The SDK refers to this concept as Tokenizer. (Internal Ticket).

  • Add a filter to the list of Documents to find Documents that need to be revised by humans. (Internal Ticket).

  • Improved training time of Extraction AIs when using character detection mode (reduced up to 40%) (Internal Ticket).

  • Train custom Page-based Categorization AI via SDK (Internal Ticket).

  • First stable release of Capture Vue in version 0.1.0.

  • Suggest page breaks if one file contains multiple Documents (Internal Ticket).

  • Redirect https://app.konfuzio.com/api/ to stable API Version 3 (Internal Ticket).

  • Make all List Views sortable by click on column (Internal Ticket).

  • Sort Labels in Label-Sets to allow users to customize the UI per Category (Internal Ticket).

  • Delta Training, Partial Fit an exisiting classifier, so that training documents used previously can be deleted (Internal Ticket)

  • Allow administrators of Konfuzio on-premise installations to run a speedtest (Internal Ticket)

  • Start automatic AI retraining after user confirms that he has finished a annotation review (Internal Ticket)

  • Show a warning if a user works on a Document that is not assigned to him (Internal Ticket).

Next Release¶

This version uses Konfuzio Trainer in version v.0.3.22, the Konfuzio Python SDK in version v.0.1.15 and Konfuzio Capture Vue in version 0.0.9.

Added¶

  • Allow to invite Members with different Roles to Projects. Available Roles are “Reviewer” and “Manager”. All existing Members keep their current Permissions and will become Managers.

  • Superusers can define custom Roles for Members: Inviting Users can select from those Roles when inviting new Members to a Project. (Internal Ticket).

  • Add the property ‘has_multiple_top_candidates’ to the Label API V3 (Internal Ticket).

  • Add the property ‘has_multiple_annotation_sets’ to the Label Set API V3 (Internal Ticket).

  • Save feedback that there are no Annotations for a Label/Label-Set combination in a document (Internal Ticket).

  • Add the property ‘number’ to the Page API V3 (Internal Ticket).

  • Include the Label Set name in the API V3 even when no Label Set is present (Internal Ticket).

  • API v3 endpoint to sort and split Pages into Documents with different categories contained in one file (Internal Ticket).

Changed¶

  • When a user creates a new Project, this user will become the default assignee for new Documents (Internal Ticket)

  • If a user rejects an Annotation, this user is tracked in the ‘revised_by’ attribute of the Annotation (Internal Ticket).

Fixed¶

  • The numbering of Annotation Sets in the SmartView does not consider deleted Annotation Sets anymore (Internal Ticket).

  • In some situations a Project could not be deleted via API (Internal Ticket).

  • In specific scenarios, the deletion of the last remaining Annotation in a Document was not possible (Internal Ticket).

  • The SmartView did not use rotated pages due to a caching problem (Internal Ticket).

  • The arrow in the Project- and language selector was not clickable (Internal Ticket).

  • On the Annotation list Page the Category filter was not showing all Annotations (Internal Ticket).
